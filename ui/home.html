1

Install Dependencies

Install required Node.js packages:

npm install

This installs ajv (JSON validator) and busboy (file uploads).

2

Migrate to Multi-Project (One-Time)

Move existing data to projects structure:

npm run migrate

This creates projects/default/ and moves your bible, prompts, and assets there.

3

Generate Prompts Index

Create the index file for the UI:

npm run index

In Phase 1 (infrastructure only), this will be empty. That's expected!

4

Start the UI Server

Launch the web interface:

npm run serve

Open http://localhost:8000 in your browser.

5

Explore the Interface

Navigate through the UI:

  • Main Prompts UI: Browse and copy generated prompts
  • Storyboard: Upload and compare rendered videos
  • Project Selector: Switch projects or create new ones

üìå Current Phase: Phase 1 (Infrastructure Complete)
This system is ready for Phase 2: generating actual prompts and rendering assets.

Phase 2: Production Workflow

1

Generate Music with Suno

Use the Suno template to create your music track. Note the final duration and section timestamps (intro, verse, chorus, bridge, etc.).

2

Initialize Phase 2

npm run init-phase2

Interactive script captures song duration, BPM, and section timestamps. Creates beat map and shot plan scaffolds.

3

Define Canon (Bible)

Fill in bible/characters.json and bible/locations.json with your characters and locations. These are the identity anchors that ensure visual consistency.

4

Create Visual References

Generate 3 reference images for each character and location using Nano Banana. Save in reference/{type}/{ID}/ with guide.json.

5

Write Shot Intents

Fill in shot_plan.json with what happens in each shot, why it happens, which characters appear, and the camera intent.

6

Compile Prompts

Merge Canon + Shot Intent ‚Üí Final Prompts using templates. Generate A/B/C/D variations (camera only).

7

Validate & Render

npm run lint && npm run index && npm run serve

Validate prompts, generate UI index, and view in browser. Copy prompts to Kling and Nano Banana for rendering.

8

Upload to Storyboard

Drag-and-drop rendered MP4s into the Storyboard. Compare A/B/C/D variations side-by-side and select the best.

9

Final Assembly in CapCut

Import selected shots, align with music timeline, apply transitions and color grading. Export final video!

üìÅ Multi-Project Usage

‚úì Complete Data Isolation: Each project has its own bible, prompts, rendered assets, and music. No data mixing between projects.

Creating a New Project

  1. Click the "+" button next to "Current Project" in the left navigation
  2. Enter a project name and description in the wizard modal
  3. Click "Create Project"
  4. The system automatically switches to your new project

Switching Between Projects

Use the dropdown selector in the left navigation. Your selection is saved in browser localStorage and persists across sessions.

Per-Project Commands

npm run index # Generate index for 'default' project npm run index my-project # Generate index for specific project
